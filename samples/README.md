# Sample 10-K filings

This directory contains four synthetic 10-K filings used for smoke tests and the CI eval. They are generated by [`synthesize.py`](./synthesize.py) so the corpus is fully reproducible from source — no PDF is downloaded from the SEC at build time.

## Filings

| Filename | Company | Purpose |
|---|---|---|
| `ACME_10K_2024.pdf` | ACME Corporation | Primary single-filing eval target. Full sections (MD&A, Risk Factors, Financial Statements, Notes), $94.2B revenue, 6 risk factors, 3 segments. |
| `BETA_10K_2024.pdf` | BETA Corporation | Cross-company comparison target. $212.0B revenue, cloud-heavy segment mix, 5 risks. |
| `GAMMA_10K_2024.pdf` | GAMMA Corporation | Cross-company comparison target. $305.0B revenue, search/ads-dominated mix, 5 risks. |
| `garbage_10K_2024.pdf` | (none) | **SC-006 negative test.** Deliberately low-quality content (no sections, no financials, mostly noise). The pipeline's 5-dimension quality rubric must score it below 22/30, so it lands in Gold but is excluded from the Vector Search index. The eval gate confirms the agent never cites it. |

## Regenerating

```bash
.venv/bin/pip install fpdf2     # one-time
.venv/bin/python samples/synthesize.py
```

Output PDFs are committed (small, ~2-3 KB each, deterministic).

## Why synthetic, not EDGAR

- **Reproducibility**: CI doesn't depend on EDGAR availability or rate limits.
- **Predictability**: `expected_facts` in `evals/dataset.jsonl` align with the values written into the synthetic PDFs by `synthesize.py`. Real 10-Ks change between filings, breaking the eval.
- **License clarity**: synthetic content is repo-owned, no SEC content redistribution concerns.

To use real EDGAR PDFs against a deployed dev environment, drop them into the volume directly:

```bash
databricks fs cp /path/to/AAPL_10K_2024.pdf \
  dbfs:/Volumes/<catalog>/docintel_10k_dev/raw_filings/
```

The pipeline picks them up via Auto Loader (file-arrival semantics — see `resources/pipelines/doc_intel.pipeline.yml` `continuous: true`).
